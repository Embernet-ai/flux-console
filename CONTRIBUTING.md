# Contributing to Flux Console

Fireball Industries welcomes contributions to Flux Console! Whether it's bug reports, feature requests, documentation improvements, or code changes — all contributions are appreciated.

## How to Contribute

1. **Fork** the repository at [`embernet-ai/flux-console`](https://github.com/Embernet-ai/flux-console).
2. **Clone** your fork and create a feature branch from `main`.
3. **Make your changes** — follow the existing code style and conventions.
4. **Test** your changes by building the library and SPA:
   ```bash
   npm install
   ng build flux-console-lib
   ng build flux-console
   ```
5. **Commit** with a clear, descriptive message.
6. **Open a Pull Request** against `main` with a description of what changed and why.

## Reporting Issues

Open an issue at [github.com/Embernet-ai/flux-console/issues](https://github.com/Embernet-ai/flux-console/issues). Include:

- Steps to reproduce
- Expected vs. actual behavior
- Browser and Node.js version
- Screenshots if applicable
